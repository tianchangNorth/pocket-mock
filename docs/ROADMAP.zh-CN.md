# PocketMocker 项目路线图

本文档概述了 PocketMocker 的开发计划。我们的目标是将项目从一个轻量级工具发展成为一个全面、企业级的 API Mock 平台。

> **注意：** 此路线图将根据社区反馈和优先级进行调整。

## 阶段 1：核心体验增强 (v1.2.0)

- [ ] **规则分组与环境管理**
    - 引入“组”或文件夹来组织规则。
    - 增加“环境”切换功能（例如：开发环境、测试环境、错误测试场景），可即时切换规则集。
- [ ] **GraphQL 支持**
    - 在 `matcher.ts` 中实现基于 `operationName` 的匹配。
    - 在规则编辑器 UI 中添加 GraphQL 特有界面。
    - 适配 Smart Mock 生成器以支持 GraphQL 响应结构（`data`/`errors`）。
- [ ] **用户体验优化**
    - [ ] 添加“一键复制”按钮。
    - [ ] 支持“克隆规则”功能。
    - [ ] 规则列表可拖拽排序。

## 阶段 2：生态与连接 (v1.3.0)

- [ ] **高级导入/导出功能**
    - 导出当前规则为 JSON 文件。
    - 导入浏览器网络录制（HAR 文件）。
    - 导出规则为 Node.js 代码（例如，供 MSW 或单元测试使用）。
- [ ] **响应流模拟**
    - 支持模拟分块响应数据，并可配置每块数据的延迟。
    - 对于测试 AI/LLM 应用（Server-Sent Events）至关重要。
- [ ] **主题系统**
    - 引入 CSS 变量以实现完整的主题支持。

## 阶段 3：架构与全栈覆盖 (v2.0.0)

- [ ] **WebSocket Mocking**
    - 拦截 `WebSocket` 构造函数。
    - 实现基于消息内容的匹配规则（发送 A -> 回复 B）。
    - 模拟服务器发起的消息推送。
- [ ] **Service Worker 模式**
    - 提供 Service Worker 拦截模式作为可选方案（类似于 MSW）。
    - 提供更底层的拦截能力，并与现有代码隔离。
- [ ] **无头模式 (Headless Mode)**
    - 允许在 Node.js 环境（如 Jest/Vitest）中复用 PocketMocker 的规则文件进行 API 测试。
